# Lab 1: Establish a CSIRT Team

## Overview

This lab focuses on establishing a Computer Security Incident Response Team (CSIRT) within an organization. You will learn the fundamentals of CSIRT frameworks, develop incident handling procedures, and create operational playbooks.

## Objectives

- Understand CSIRT roles and responsibilities
- Design an incident response lifecycle aligned with NIST/ISO frameworks
- Create incident classification and severity matrices
- Develop standard operating procedures (SOPs)
- Build escalation and communication playbooks

## Tasks

### Task 1: Define CSIRT Structure
- Identify team roles (Incident Manager, Analyst, Forensics, Comms)
- Document responsibilities for each role
- Create an org chart for the CSIRT

### Task 2: Incident Classification Matrix
- Define severity levels (P1-Critical to P4-Low)
- Map incident types to severity levels
- Create a decision tree for incident triage

### Task 3: Incident Response Playbooks
- Develop a phishing incident playbook
- Develop a malware outbreak playbook
- Develop a data breach escalation playbook

### Task 4: Documentation Templates
- Incident report template
- Post-incident review (PIR) template
- Evidence chain of custody form

## Deliverables

- CSIRT organizational chart
- Incident severity matrix
- Three incident response playbooks
- Documentation templates (incident report, PIR, chain of custody)

## Skills Gained

- CSIRT framework design
- Incident classification and triage
- Playbook development
- Stakeholder communication planning

- ISO/IEC 27035 - Information Security Incident Management
